# Patterns — John

## Kända mönster (läs INNAN du börjar)

### ⚠️ Exit code 0 ≠ uppgift klar
Verifiera ALLTID med `git log --oneline -1`. Är commit-meddelandet kopplat till uppgiften? Om nej — kör om.

### ⚠️ Aider ändrar fel fil
Om du ger --file X men Aider ändrar Y → omformulera uppgiften, var mer explicit om exakt vad som ska ändras i filen.

### ⚠️ HEARTBEAT_OK med kort i kön
Om QUEUE.md har kort i Ready → ta dem. Svara ALDRIG HEARTBEAT_OK när det finns arbete att göra.
